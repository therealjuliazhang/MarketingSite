Our Project

Dr Holly Tootell’s research project aims to investigate the design process of the Jump Start application, which is a development tool on physical activity and gross motor skills for pre-schoolers and parents and educators. In order to achieve this, various tests have been conducted on paper, with results written down by hand.

Our team’s job is to make the testing and data collection process easier for the researchers.

Our proposed solution to this problem is an online web application that aims to make it convenient and efficient to conduct the tests. By using a web application we can have the tests conducetd on devices such as tablets and automatically collect data and compile results into useful graphs and tables. A web application also allows for easy deplyment to preschools and allows for instructions to be displayed through our application to inform educators on how to conducts tests.

Our Solution

Methodology

Data Collection
Currently, the tests are conducted on paper, with results written down by hand and then entered into a spreadsheet. This causes difficulty in data collection as every result must go through this time-consuming process. This limits the effectiveness of how many tests can be conducted within the given time frame. There is also potential for accuracy issues in data collection due to human error.

Deployability
Because the tests are conducted on paper, during each testing 'round', extensive preparation must be made to ensure each educator has the correct papers for each task of the tests.
Our solution, Child’sPlay, aims to minimise the troubles involved in launching paper tests. This includes digitising data collection and deploying tests on a web application to allow for accessibility across various devices. In addition, tests can be created and saved into Child’sPlay for later use.

Agile
Our methodology was based off the agile methodology scrum. Weekly sprints with meetings twice a week were held throughout the project. Our methodology differed from scrum as we did not strictly follow scrum's structure in having daily face-to-face meetings. In taking a similar approach to scrum, several tasks can be assigned to several members, and we are able to respond to changes and adapt to new requirements.

Taiga.io
Using Taiga allows for sprint planning for our agile methodology and also enables the team to manage the tasks that need to be done. It also allows for the team to update where they are up to on a task.
